# Conventions

import { Callout } from "nextra-theme-docs";

In order to maintain consistency across the code base, we have converged on a few conventions.

<Callout>Conventions are meant to be challenged and improved!</Callout>

## Components

We use a few different file naming conventions across projects. The most common naming convention is for a component.

<Callout>
  All of these are not always necessary, but provide a framwework for how we
  like to break up code.
</Callout>

- **`Component.tsx`** - the React component
- **`Component.definitions.ts`** - the type definitions of props and state
- **`Component.styles.tsx`** - the `@emotion/styled` components used by the component
- **`Component.utils.ts`** - any utility functions used by the component
- **`index.ts`** - the export of the component

import { CounterProps } from "./Counter.definitions";
import { Count, DecrementButton, IncrementButton } from "./Counter.styles";

const Counter = ({ value, onChange }: CounterProps) => {
  return (
    <Count>{value}</Count>
    <DecrementButton onClick={() => onChange(value-1)}>-1</DecrementButton>
    <IncrementButton onClick={() => onChange(value+1)}>+1</IncrementButton>
  );
};
